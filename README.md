# Hotel Reviews — Accommodation Management System

Data analysis and machine learning on the [515K Hotel Reviews Data in Europe](https://www.kaggle.com/datasets/jiashenliu/515k-hotel-reviews-data-in-europe) dataset (Kaggle), aiming to build the MVP of a web platform for hotel managers: analyzing historical reviews, extracting actionable insights, and predicting/explaining the hotel's average rating.

## Project structure

```
data/                          # raw dataset (not versioned — see "Data" below)
notebooks/
  exploratory/                 # exploratory analysis notebooks
  nlp_pipeline_colab.ipynb     # self-contained NLP pipeline for Google Colab (no local GPU needed)
src/                           # modular Python pipeline
  data_quality.py                 # Phase 1 — dataset quality audit
  panel_feasibility.py            # statistical feasibility of the hotel x period panel
  text_cleaning.py                # review text diagnostics and normalization
  nlp_pipeline.py                 # overall + aspect-based sentiment extraction (local GPU)
outputs/                       # generated artifacts (parquet, csv) — not versioned (except sample)
figures/                       # charts generated by the analyses
reports/                       # Markdown reports from audits and run guides
```

The raw CSV (`Hotel_Reviews.csv`, ~230MB) **is not versioned** in the repository (it exceeds GitHub's file size limit). Download it from Kaggle and place it at `data/Hotel_Reviews.csv` before running any script.

## NLP pipeline

Enriches every review with **overall sentiment** (1–5 stars) and, per aspect (cleanliness, staff, location, comfort, food, value for money, noise, facilities), **three columns**:

- `aspect_<name>_score` — a 0–10 sentiment score (0 = most negative, 10 = most positive; empty when the aspect isn't mentioned)
- `aspect_<name>_subtags` — closed-vocabulary tags (e.g. `water_quality`, `wifi`, `rudeness`) so the same problem is **comparable and countable across reviews and hotels**, not just free text
- `aspect_<name>_evidence` — a verbatim excerpt from the review (audit trail only, never generated/invented)

Uses free, local models — no data leaves the machine — (`nlptown/bert-base-multilingual-uncased-sentiment` for overall sentiment, `yangheng/deberta-v3-base-absa-v1.1` for aspects). Two ways to run it:

**Option A — local machine with a CUDA GPU.** Full guide: [`reports/RUN_NLP_ON_GPU.md`](reports/RUN_NLP_ON_GPU.md). Summary:

```bash
python src/nlp_pipeline.py --sample 500 --batch-size 64   # smoke test — always run first
python src/nlp_pipeline.py --batch-size 128                 # full run
```

**Option B — Google Colab (free T4 GPU, no local setup).** Open [`notebooks/nlp_pipeline_colab.ipynb`](notebooks/nlp_pipeline_colab.ipynb) in Colab (`Runtime → Change runtime type → T4 GPU`). Self-contained — no dependency on this repo's files, only needs `Hotel_Reviews.csv` uploaded to Google Drive. Runs in chunks with resumable checkpoints saved to Drive, so a Colab disconnect never loses more than one chunk.

Either path produces `outputs/reviews_enriched.parquet`.

## Web app

The `web/` folder is the TrueStay front-end (React + Vite), currently running on mock data. See
[`web/README.md`](web/README.md) for how to run it locally — short version:

```bash
cd web
npm install
npm run dev
```

Then open http://localhost:5173.
